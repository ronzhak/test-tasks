# Задание 1: Анализ требований
Найдите и перечислите все логические противоречия и недочеты в предоставленном ТЗ. Объясните, в чем заключается проблема для каждого пункта.

1. Пользователь может добавить в корзину от 1 до 10 единиц одного товара.
2. Пользователь может изменить количество каждого товара в корзине не менее, чем до 1-го. Для удаления товара из корзины используется отдельная кнопка.
3. В корзине может находиться не более 5 различных товаров.
> Проблема: Фраза «...не более 5 различных товаров» имеет неоднозначность, т. к. может быть воспринята как ограничение по количеству товаров в корзине.

> Решение: `В корзине может находиться не более 5 видов различных товаров.`
4. Суммарное количество всех товаров в корзине не может превышать 20 штук.
5. Товары в корзине могут быть разные.
> Проблема: Требование не отвечает требованию полноты и нуждается в дополнительной детализации. Указать какие отличия в характеристиках товара, следует считать различием. Например размер/цвет/принт и т.п.

> Решение: `Товары в корзине могут быть разных размеров, цветов, комплектаций.`
6. При попытке добавить товар, превышающий лимиты, система показывает сообщение: "Лимит корзины превышен".
> Проблема: текст сообщения не достаточно детализирован, т. к. используется 3 вида лимитов, непонятно над каким именно нужно поработать.

> Решение: Формировать текст сообщения для каждого вида лимита:<br/>
    a) `При попытке добавить товар, превышающий лимит по количеству единиц одного товара, система показывает сообщение: Превышен лимит по количеству единиц одного товара`.<br/>
    b) `При попытке добавить товар, превышающий лимит всех товаров в корзине, система показывает сообщение: Превышен лимит количества товаров для заказа. Оно не должно превышать 20 шт.`<br/>
7. Цена на продукт фиксируется на момент добавления в корзину и не меняется.
8. На странице корзины отображается список товаров, их количество, цена за единицу и общая стоимость позиции.
9. Если пользователь уменьшает количество товара до 0, товар удаляется из корзины.
> Проблема: противоречие с п.1 и 2. Товаров не может быть меньше 1. Для удаления используется отдельная кнопка.

> Решение: <br/>
    a) `Если пользователь уменьшает количество товара до 0, автоматически удалять его из заказа.`<br/>
    b) `Пользователь не может уменьшить количество товара до 0.`
10. В корзине может быть реклама других продуктов.
> Проблема: требование недостаточно детализировано.

> Решение: детализировать функциональность рекламы других продуктов.
>- формат рекламы (баннер/мини-карточки товаров)
>- расположение на странице
>- количество
>- оформление
>- отображаемые характеристики: цена, срок доставки, «похожесть»
>- возможность добавления к заказу
>- просмотр дополнительной информации о товаре на этой же странице
>- есть ли возможность скрыть рекламу

> `В корзине может быть реклама других продуктов. Рекламный блок располагается внизу страницы перед кнопкой Оформить заказ, и представляет собой карусель 10-ти изображений товаров с ценами.
`

11. Реклама товаров в корзине должна быть каждый будний день по утрам и вечерам.
> Проблема: формулировка «по утрам и вечерам» не имеет четких границ.

> Решение: `Реклама товаров в корзине должна быть каждый будний день по утрам c 9:00 до 11:00 и вечерам c 17:00 до 19:00 по часовому поясу клиента.`
13. Если цена на товар изменилась в каталоге, система должна автоматически обновить ее в корзине у всех пользователей.
> Проблема: формулировка противоречит п. 7.
> Решение:<br/>
    a) `Если цена на товар изменилась в каталоге, то в корзине пользователя она остаётся неизменной.` <br/>
    b) `Если цена на товар изменилась в каталоге, то уведомить об этом пользователя при оформлении заказа.`

 - g) Может ли меняться период отображения (дни, время)?
